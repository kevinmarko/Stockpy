# Architecture: Observability & Read-Only APIs

> Part of the split-out `CLAUDE.md` System Architecture reference. See [../../CLAUDE.md](../../CLAUDE.md) for the index.

- **observability/__init__.py** — Empty package marker.
- **observability/alerts.py** — `send_alert(level, message, channels=None, extra=None) -> None` dispatches to zero or more output channels: `console` (always active via Python logging), `file` (JSONL at `settings.ALERT_FILE_PATH`), `discord` (`settings.DISCORD_WEBHOOK_URL`), `slack` (`settings.SLACK_WEBHOOK_URL`), `email` (SMTP via `settings.ALERT_SMTP_*`). `_active_channels()` auto-detects configured channels. Discord payload: `{"content": "🚨/⚠️/ℹ️ **[LEVEL]** \`timestamp\`\nmessage"}`. Slack payload: `{"text": "..."}`. Email: MIMEText via `smtplib.SMTP` STARTTLS. All channel failures are caught and logged — a broken webhook must never crash the pipeline. Alert rules: CRITICAL for kill switch, reconciliation drift, broker lost, missing validation report; WARNING for heat >5%, correlation concentration, large slippage; INFO for fills, rebalance. `send_daily_summary(pnl_summary, warnings)` composes a multi-line INFO alert with P&L and warnings.
- **observability/dashboard.py** — **Retired.** The standalone Streamlit paper-trading observability dashboard has been removed; its panels are folded into the Command Center's Observability tab (`gui/panels/observability.py`) so there is a single always-on surface instead of two separately-launched Streamlit apps.
- **api/state_api.py** — **Standalone, read-only FastAPI service** over the file-backed state the engine and UI already communicate through — a foundation for a future web/mobile frontend, **deliberately NOT wired into any orchestrator, the desktop shell, or the GUI**. Endpoints: `GET /health` (liveness of the API process, ALWAYS open — no auth — so a load balancer/watchdog can probe without a token), `GET /state` (full parsed `output/state_snapshot.json`, 404 if absent), `GET /signals` (just the `signals` list from that snapshot), `GET /trades` (closed trades via `transactions_store.TransactionsStore`, `[]` when none). **AST-guard invariant:** it only reads already-persisted state and NEVER imports engine/calculation modules (`processing_engine`, `strategy_engine`, `forecasting_engine`, `macro_engine`, …) or broker/execution modules (`execution/*`). **CORS** via `CORSMiddleware` restricts browser origins to `settings.CORS_ALLOWED_ORIGINS` (GET-only, `allow_credentials=False`). **Bearer-token auth** (`require_token` via built-in `HTTPBearer`): when `settings.STATE_API_TOKEN` is set, `/state`/`/signals`/`/trades` require `Authorization: Bearer <token>` (constant-time compare via `hmac.compare_digest`, 401 on mismatch/missing); `/health` is always open. **FAIL-OPEN** when `STATE_API_TOKEN` is unset/empty (the three data endpoints are unauthenticated for zero-config local use, with a startup WARNING logged). The token is NEVER logged (CONSTRAINT #3). Dead-letter resilient (CONSTRAINT #6): a missing/unreadable/invalid snapshot degrades to `None`→404 and a DB read failure degrades to `[]` — it never fabricates a placeholder/synthetic snapshot (CONSTRAINT #4). Launch standalone: `uvicorn api.state_api:app --port 8600` (no new dependencies — fastapi/uvicorn already in requirements.txt, `HTTPBearer` is built into fastapi). **New env vars:** `STATE_API_TOKEN` (secret bearer token; when set, `/state` `/signals` `/trades` require `Authorization: Bearer <token>`, 401 otherwise; unset = fail-open for local use), `CORS_ALLOWED_ORIGINS` (JSON list of allowed browser origins, default `["http://localhost:3000"]`).
- **api/control_api.py** — **Separate FastAPI app from `api/state_api.py`** (deliberately — `state_api.py`'s read-only purity is AST-guard-enforced to never import engine/execution modules; this module's whole purpose is to reach the daemon and the kill switch, so it needs its own file and its own auth posture). Hosted INSIDE the orchestrator daemon process by `desktop/orchestrator_daemon.py` (not standalone-launchable the way `state_api.py` is). `set_daemon(daemon)`/`get_daemon()` module-level registry, wired by the daemon entrypoint after `daemon.start()`. Endpoints: `GET /health` (always open — `{"status": "ok", "daemon_alive": bool}`); `GET /status` (fail-open read token, mirrors `state_api.py`'s `STATE_API_TOKEN` posture — daemon status, `last_run`, kill-switch state, `advisory_only`, `dry_run`); `POST /run` (**FAIL-CLOSED** command token via `settings.ORCHESTRATOR_DAEMON_TOKEN` — unlike `STATE_API_TOKEN`, unset means the endpoint is DISABLED entirely (403), not open, since triggering a real pipeline run is a materially different risk than reading persisted state; checks auth first, then `execution.kill_switch.GlobalKillSwitch` — 423 if active — then `daemon.trigger_run(reason="manual")` — 202 + run_id on success, 409 + the existing run_id if one is already in flight); `GET /run/{run_id}/status` (200 with `state: "running"` for an in-flight run — NOT a 404 — or 404 for a truly unknown id); `GET /run/latest` (404 only when nothing has ever been triggered); **`PUT /interval`** (2026-07, live interval setter — command-token guarded ALONE, same posture as `POST /run`, since a live setter has no persistence to protect unlike `.env`; pydantic body validated via the shared `settings.validate_interval_seconds`; calls `daemon.set_interval(body.interval_seconds)` and echoes `{"interval_seconds": ...}` — see `desktop/daemon_runtime.py` above); **`GET /runs/history`** (2026-07, new — durable run history, read-token guarded like `GET /status`; reads `desktop/run_history_store.py`'s `pipeline_runs` DB table via `RunHistoryStore(readonly=True).get_recent(limit)`, `limit` query param clamped to `[1, 200]` (default 50), degrades to `[]` on a DB read failure rather than a 500; deliberately has NO daemon-not-attached branch — unlike every other endpoint here, the whole point is to keep serving history across a daemon restart, exactly when `GET /status`'s in-memory `run_history` ring is empty). CORS mirrors `state_api.py` but adds `POST`/`PUT` to `allow_methods`. - **llm/status_store.py** (2026-07, new) — **Last-real-call telemetry for the LLM providers**, closing the "a missing OR invalid key degrades analyst narratives silently to `null`" gap (§4 of `docs/project_review_and_gap_analysis.md`). Presence checks alone couldn't close it: `llm/providers.py` swallows a 401 into the identical `None` as a timeout, so a garbage-but-present key showed 🟢 ready and still produced `null`. Instead of a live probe (which would spend the operator's money making an LLM call just to test a key, on a fail-open endpoint), this records what actually happened on the last **real** call, written from `llm/providers.py`'s own except blocks: `record_failure(provider, exc)` in each call-except, `record_success(provider)` the instant the SDK call returns (BEFORE parsing, so every downstream `return None` — missing block, refusal, `parsed=None` — still clears a stale auth verdict), and `record_failure(..., error_kind="schema")` in the parse handlers. Public API: `classify_exception(exc) -> (error_kind, http_status)`, `record_success`/`record_failure`, `read_status(provider)`/`read_all()`, `clear()`. **Classification imports NO SDK** — by `type(exc).__name__` + an HTTP status read off the exception object (`data/robinhood_portfolio.verify_credentials`'s "only exception type names" convention); anthropic/openai share Stainless class names (`AuthenticationError`→auth, `RateLimitError`→rate_limit, …), while google.genai has NO auth class — a bad key is `ClientError(code=400, "API key not valid")`, matched by an exact documented-reason regex; **every other 400 stays `unknown`, never a guessed auth**. **The honesty rule:** we never claim "your key is invalid *now*", only "the last real call was rejected: authentication" (past-tense, timestamped, true forever) — so all UI copy is timestamped and only `error_kind=="auth"` ever renders as `invalid_key` (a rate-limit/network/timeout is telemetry, never a key problem). **Two staleness bounds, each scoped to its claim:** key-identity verdicts (`ok`/`auth`) are bounded by a truncated one-way SHA-256 **fingerprint** of the key — a verdict whose fingerprint ≠ the CURRENT key is discarded as `source="key_rotated"` with every field nulled, so **fixing a key clears a false alarm INSTANTLY, with zero LLM calls** (the whole payoff); transient verdicts are bounded by `settings.LLM_STATUS_MAX_AGE_HOURS` (default 24, `source="expired"`, fields RETAINED). **Secret containment (CONSTRAINT #3):** the KEY value is never persisted/logged; the fingerprint is MODULE-PRIVATE — `read_status`/`read_all` strip it, so it cannot reach an API response by construction. **Leaf invariant:** imports nothing from the `llm` package (cycle-proof, so `from llm import status_store` runs safely from inside `llm/providers.py`). Atomic write-then-`os.replace` (mirrors `llm/cache.py::_write_all`) under a module `threading.Lock`; always writes (advances `checked_at` so the age bound reflects last-observed); **never raises** (CONSTRAINT #6) — `llm/providers.py` calls it BARE (no call-site wrapping), relying on the store's own non-raising guarantee, so a telemetry write can never break the analyst-commentary soft-fail contract.
