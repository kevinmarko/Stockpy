# Architecture: Data Layer & Persistence

> Part of the split-out `CLAUDE.md` System Architecture reference. See [../../CLAUDE.md](../../CLAUDE.md) for the index.

- **config.py** ‚Äî Single Source of Truth (SSOT). `COLUMN_SCHEMA` defines every column's Google Sheets header, internal dict key, and display format, plus Pandera schemas for validation. Any new field must be added here first before use elsewhere.
- **dto_models.py** ‚Äî `MarketBarDTO`, `FundamentalDataDTO`, `MacroEconomicDTO`, `RobinhoodPositionDTO`. All market/fundamental/macro data flowing through calculation layers must be coerced into these DTOs ‚Äî raw dict lookups or positional indexing in calc code is not allowed. DTO parsers also handle messy upstream strings (currency symbols, `%`, `"N/A"`, padding). `FundamentalDataDTO` carries a `current_ratio` field (`__init__` default `NaN`; `from_raw_dict` reads `info.get("currentRatio", NaN)`) ‚Äî `currentRatio` is already emitted by `data/yahoo_fundamentals.py` and is now carried through the DTO and surfaced by `engine/advisory.py` as `key_indicators["current_ratio"]`. `MacroEconomicDTO` now accepts an optional `hmm_risk_on_probability: Optional[float] = None` (the `regime/hmm_regime.py` second opinion). `None` (the default) reproduces the exact pre-HMM behavior. The rules-based `_rules_based_regime` stays primary; `market_regime` downgrades RISK ON ‚Üí NEUTRAL (logged) when `hmm_risk_on_probability < HMM_RISK_ON_DOWNGRADE_THRESHOLD` (0.3) ‚Äî it can never independently declare a worse regime or upgrade a better one. `killSwitch` ORs in a lowered-threshold check (`vix > KILLSWITCH_VIX_THRESHOLD_AGREED`=25 or `sahm_rule_indicator >= KILLSWITCH_SAHM_THRESHOLD_AGREED`=0.3, vs. the base 30/0.5) only when the rules-based regime is RECESSION AND the HMM agrees (`1 - hmm_risk_on_probability > HMM_RISK_OFF_AGREEMENT_THRESHOLD`=0.7) ‚Äî strictly OR'd with the base condition, never less sensitive.
- **data_engine.py** ‚Äî `DataEngine` (live Yahoo Finance/FRED via `fredapi`) and `MockDataEngine` (test fixture), both implementing an `IDataProvider` interface. Fetching is strictly decoupled from calculation. `IDataProvider` now also declares `fetch_macro_history() -> pd.DataFrame` (full historical VIX/yield-curve series for `regime/hmm_regime.py`'s expanding-window fit, distinct from `fetch_macro_raw()`'s single current-snapshot dict). `DataEngine.fetch_macro_history()` pulls full unbounded series via `self.fred.get_series('VIXCLS'/'T10Y2Y')`; returns an empty DataFrame (never fabricated rows) if FRED is unavailable. `MockDataEngine.fetch_macro_history()` returns a deterministic 500-row synthetic series (seeded) for offline tests. **Parallel fetch (2026-07):** `DataEngine.fetch_technical_raw()`/`fetch_fundamentals_raw()` were serial `for symbol in tickers:` loops making one blocking yfinance HTTP call at a time; both now route through a `ThreadPoolExecutor` sized by `settings.DATA_FETCH_MAX_CONCURRENCY` (default 8, matching the `FORECAST_MAX_CONCURRENCY`/`ADVISORY_MAX_CONCURRENCY` convention), collapsing wall-clock fetch time from ~N sequential round-trips to ~N/workers. Per-ticker try/except is preserved (one bad symbol still can't abort the batch); the old `sleep(0.1)`-every-5-tickers throttle in `fetch_fundamentals_raw` was dropped in favor of the bounded worker count as the de-facto rate limit. Set `DATA_FETCH_MAX_CONCURRENCY=1` to force the original sequential path. **New env var:** `DATA_FETCH_MAX_CONCURRENCY`. **`fetch_technical_raw_cached()` ‚Äî HistoricalStore routing for `main_orchestrator.py` (2026-07):** `main_orchestrator.py`'s `fetch_all_data_async()` was the last major tech-bars call site that bypassed `HistoricalStore` entirely ‚Äî it called `DataEngine.fetch_technical_raw()` directly every cycle for the full ticker universe (unlike `main.py`'s `_fetch_bars_for_universe()`, which already routes through `HistoricalStore.get_bars()`). `DataEngine.fetch_technical_raw_cached(tickers)` closes this gap: a new, additive method (alongside, not replacing, `fetch_technical_raw()`) that lazily imports `data.historical_store.HistoricalStore` and `data.market_data.get_provider()` (this repo's `HistoricalStore`-lazy-import convention, matching `processing_engine.py`/`macro_engine.py`) and routes each ticker through `HistoricalStore.get_bars(symbol, lookback_days=settings.BARS_BACKFILL_DAYS, provider=<CompositeProvider singleton>)` instead of a fresh `yf.Ticker(symbol).history(period="2y")` call ‚Äî a symbol whose bars are already persisted needs only its `(last_date, today]` delta fetched. Preserves the exact `ThreadPoolExecutor`/`DATA_FETCH_MAX_CONCURRENCY` concurrency pattern and per-ticker try/except dead-letter resilience of `fetch_technical_raw()`. Falls through to calling `fetch_technical_raw(tickers)` verbatim (byte-identical output) when `settings.HISTORICAL_STORE_ENABLED` is `False`, or when `HistoricalStore`/`get_provider()` construction raises ‚Äî CONSTRAINT #6, this can only ever remove a redundant refetch, never silently substitute bad data. `MockDataEngine` gained a matching `fetch_technical_raw_cached()` that is a plain alias for its own `fetch_technical_raw()` (mock data is synthesized fresh from `datetime.now()` on every call, so there is no real per-ticker fetch to cache against a DB; this also keeps `main_orchestrator.py`'s existing `MockDataEngine()` offline-fallback branches ‚Äî no `credentials.json`, or a live-fetch coming back empty ‚Äî working unchanged now that the call site is unconditional). `fetch_technical_raw()` itself is unchanged.
- **data/market_data.py** ‚Äî Swappable market-data layer with a provider abstraction (`MarketDataProvider` ABC) that hides the concrete backend from all signal, indicator, and forecasting code. **Provider auto-selection** (evaluated at `CompositeProvider` construction): (1) `MARKET_DATA_PROVIDER=alpaca` or Alpaca keys present ‚Üí `AlpacaProvider` (real-time IEX via alpaca-py); (2) otherwise ‚Üí `YFinanceProvider` (zero config, ~15-min delayed, `is_stale=True` always). **Components**: `Quote` (frozen dataclass: symbol, price, bid, ask, UTC timestamp, is_stale, source); `MarketDataError` (typed exception ‚Äî orchestrator catches per-symbol for dead-letter resilience); `AlpacaProvider` (`StockHistoricalDataClient` + `StockLatestQuoteRequest` + `StockBarsRequest`, IEX feed, stale if quote age > `stale_threshold_seconds`=60); `YFinanceProvider` (`Ticker.fast_info` for quotes, `Ticker.history()` for bars ‚Äî always `is_stale=True`, wraps yfinance errors as `MarketDataError`; ALSO the fundamentals fallback, returning raw `Ticker.info` when the primary Yahoo-computed engine yields nothing); `YahooFundamentalsProvider` (`SOURCE="yahoo_computed"` ‚Äî the **PRIMARY fundamentals source**; pulls Yahoo's FREE financial statements ‚Äî `income_stmt`, `balance_sheet`, `cashflow`, `dividends`, `institutional_holders` ‚Äî plus price/shares, then delegates to the pure `data/yahoo_fundamentals.py::compute_fundamentals(...)` to derive 15 equity fundamentals as a yfinance `.info`-style dict so `FundamentalDataDTO.from_raw_dict()` is unchanged; NaN-degrading per CONSTRAINT #4 ‚Äî missing input ‚Üí NaN, never a fabricated 0.0; caches SPY returns for the beta computation); `FinnhubProvider` (**DEPRECATED as a fundamentals source and UNWIRED from `CompositeProvider`** ‚Äî the class is retained but no longer consulted for fundamentals; `FINNHUB_API_KEY` is now used ONLY by `signals/news_catalyst.py` for company news / earnings); `CompositeProvider` (routes quotes/bars to the selected backend, fundamentals to **`YahooFundamentalsProvider` (primary) ‚Üí yfinance `.info` (emergency fallback)** ‚Äî Finnhub is no longer in the fundamentals path; exposes a `source_name` property reporting the active fundamentals backend; in-process TTL quote cache, logs startup banner once). **Bar shape contract**: `get_intraday_bars()` returns a DataFrame with columns `Open, High, Low, Close, Volume` and a timezone-naive `DatetimeIndex` sorted ascending ‚Äî identical to `DataEngine.fetch_technical_raw()`'s shape so all processing/forecasting/strategy code runs unchanged. **Quote cache**: in-process dict + monotonic TTL (default `MARKET_DATA_QUOTE_TTL_SECONDS`=30 s), never written to disk. **Bars cache (2026-07, PR A hot-path)**: `CompositeProvider.get_intraday_bars()` also carries an in-process `_BarsCache` (thread-safe `threading.Lock`, keyed `(symbol, lookback_days)` ‚Üí `(df, monotonic-expiry)`, default `MARKET_DATA_BARS_TTL_SECONDS`=300 s, returns a defensive `df.copy()` so callers can't poison the cache) so the "bars fetched twice per symbol" pattern (universe pre-fetch + advisory-overlay refetch) collapses to one backend pull within the TTL ‚Äî bars are daily-resolution so a few-minutes TTL is safe. `invalidate_bars(symbol, lookback_days)` / `clear_bars_cache()` force a refresh. **Two SCALE-CRITICAL emission rules (do NOT "fix" ‚Äî downstream depends on them)**: `dividendYield` is emitted as a **FRACTION** (e.g. `0.0257`, not `2.57`) ‚Äî it is NOT routed through `normalize_yfinance_dividend_yield` and is consumed as-is; `debtToEquity` is emitted as a **PERCENT** (`√ó100` ‚Üí e.g. `150.0`, not `1.5`) because two downstream consumers divide by 100. Plain-fraction ratios (`returnOnEquity`, `grossMargins`, `operatingMargins`, `payoutRatio`) are unscaled; `payoutRatio` uses `abs()` of the (negative) "Cash Dividends Paid" cash-flow line. **Beta** = `Cov(stock, SPY) / Var(SPY)` over `BETA_LOOKBACK_DAYS` (default 504) of daily returns, with the SPY return series cached inside the provider. **Fundamentals cache**: `CompositeProvider` carries an in-process `_FundamentalsCache` (default `FUNDAMENTALS_CACHE_TTL_SECONDS`=21600 s / 6 h) that caches positive AND empty (negative) responses so coverage-miss symbols never re-hit the network within the TTL window. **Finnhub 429 machinery (retained but dormant for fundamentals)**: the deprecated `FinnhubProvider` still wraps outbound calls in a `_SlidingWindowRateLimiter` (default `FINNHUB_RATE_LIMIT_PER_MIN`=50) with one-shot exponential backoff + 429 retry ‚Äî relevant now only if a caller invokes it directly, since it is no longer on the fundamentals path. `clear_fundamentals_cache()` purges the cache for forced refresh. Module-level singleton via `get_provider()` / `reset_provider()`. **New env vars**: `MARKET_DATA_PROVIDER`, `FUNDAMENTALS_SOURCE` (`"yahoo"` default | `"yfinance_info"`), `BETA_LOOKBACK_DAYS`, `FINNHUB_API_KEY` (now `news_catalyst`-only, NOT a fundamentals dependency), `MARKET_DATA_QUOTE_TTL_SECONDS`, `MARKET_DATA_BARS_TTL_SECONDS` (bars-cache TTL, default 300 s, GUI-writable non-secret allowlist key), `FUNDAMENTALS_CACHE_TTL_SECONDS`, `FINNHUB_RATE_LIMIT_PER_MIN`.
- **data/sentiment_sources.py** (Sentiment Pipeline Phase 3, 2026-07) ‚Äî Multi-source sentiment-ingestion provider abstraction, structurally mirroring `data/market_data.py`'s `MarketDataProvider` ABC but **fanning out** across every enabled source rather than selecting one backend, since each source contributes independent documents. **Master switch: `settings.SENTIMENT_INGESTION_ENABLED`, default `False`** ‚Äî the sole call site, `signals/news_catalyst.py::NewsCatalystSignal._run_multi_source_ingestion()`, is a complete no-op (no network call attempted for any symbol) until an operator sets this `True`; needed because two sources (Yahoo RSS, GDELT) take no API key and so have no other way to stay quiet by default the way Finnhub/Reddit/EDGAR already do via absent credentials ‚Äî this also matches the codebase's existing convention for opt-in networked features (`ORCHESTRATOR_DAEMON_ENABLED`, `GRAVITY_AI_RUNNER_ENABLED`, `PILOTS_API_ENABLED` all default `False` the same way). `SentimentSource` ABC (`fetch(symbol, since) -> List[SentimentDocument]`, must return `[]` never raise); concrete free-first providers `FinnhubSentimentSource` (wraps `signals/news_catalyst.py`'s existing client), `YahooRSSSource`, `GDELTSource`, `GoogleNewsRSSSource`, `RedditSource` (OAuth2 client-credentials, degrades to `[]` without `REDDIT_CLIENT_ID`/`REDDIT_CLIENT_SECRET`), `EdgarSource` (SEC 8-K filings by CIK lookup -- always-on submissions-feed headlines, degrades to `[]` without a compliant `EDGAR_USER_AGENT` -- **plus an opt-in EDGAR full-text-search (EFTS) extension**, see below). **`GoogleNewsRSSSource` (`"google_news"`, opt-in, NOT in the `SENTIMENT_SOURCES` default)**: queries `https://news.google.com/rss/search?q=...&hl=en-US&gl=US&ceid=US:en`, appending Google's own `when:` search operator (`settings.GOOGLE_NEWS_LOOKBACK_WINDOW`, default `"7d"`) to the query string; parsed via `BeautifulSoup(resp.content, "xml")`, same convention as `YahooRSSSource`. **Query simplification caveat**: rather than resolving a bare ticker to its company name (a new API dependency), the query is constructed as `f"{symbol} stock"` ‚Äî a simple, dependency-free disambiguator, not a precise company-name match; documented on the class, not hidden. **Opaque-link caveat**: every `<link>` in the response is an opaque `news.google.com/rss/articles/...` redirect token, not the publisher's real URL, which defeats URL-based dedup entirely ‚Äî `GoogleNewsRSSSource` closes that gap itself with a pre-return fuzzy-title dedup (`_fuzzy_dedup`, Jaccard token-overlap on normalized titles) before the composite's own exact-text dedup ever runs. Capped at ~100 items (`_MAX_ITEMS`) ‚Äî Google's own feed limit, no pagination exists. Batch-scores every headline from one `fetch()` call through `signals.news_catalyst.score_headlines()` (PR417's batched entry point) in a single call rather than looping single-headline scoring. Enable by adding `"google_news"` to `SENTIMENT_SOURCES` in `.env`. `SentimentDocument` (frozen dataclass) carries optional credibility-relevant raw inputs (`author_handle`, `author_followers`, `account_age_days`) ‚Äî `None` when the source doesn't expose them, never fabricated (CONSTRAINT #4). `CompositeSentimentSource` fans out per `settings.SENTIMENT_SOURCES` (comma-separated, default `"yahoo_rss,gdelt,reddit,edgar"` ‚Äî `finnhub` deliberately excluded since `NewsCatalystSignal.pre_compute()` already fetches it directly every cycle; adding it here too would double-fetch), dedupes via a rolling hash keyed on `(source, symbol, trading_day, text)`, applies the optional `desentencize()` FinBERT preprocessing trick (period‚Üísemicolon, protecting decimals and abbreviations; gated by `SENTIMENT_DESENTENCIZE_ENABLED`, default `False`), and enforces a per-cycle document budget (`SENTIMENT_MAX_DOCUMENTS_PER_CYCLE`, default 2000) as its bounded, in-process backpressure mechanism ‚Äî sources are polled in priority order (`finnhub, edgar, yahoo_rss, gdelt, google_news, reddit`) and lower-priority sources are shed once the budget is hit, **never** touching order/broker code under any pressure condition. Two further bounds close the gap a per-request `timeout` alone leaves open: a **wall-clock ceiling** (`SENTIMENT_INGESTION_MAX_SECONDS_PER_CYCLE`, default 60s, tracked from `reset_cycle()`) that skips ingestion entirely for the rest of a cycle once elapsed, and a **per-source circuit breaker** (`SENTIMENT_CIRCUIT_BREAKER_THRESHOLD`, default 3 consecutive failures) that stops re-attempting a source that's clearly down instead of re-timing-out on it for every remaining symbol ‚Äî added 2026-07 after a live run against an unreachable host (GDELT) hung the pipeline for 49+ minutes with no ceiling in place. `fetch_and_archive()` scores every batch via `signals/credibility.py::score_documents()` (Phase 4 ‚Äî `S_authority`/`S_humanity`/`S_verification` sub-scores ‚Üí a `credibility_weight` in `[0.1, 1.0]`; institutional sources are policy-trusted at `1.0`, social sources scored from follower count + a batched per-author posting-cadence statistic) then persists every document to `data/historical_store.py`'s `sentiment_ingestion_audit` table (gated by `SENTIMENT_AUDIT_ENABLED`) via the shared `HistoricalStore.resolve_trading_day()` UTC‚ÜíET post-close roll. `HistoricalStore.get_sentiment_aggregate_by_symbol(trading_day)` is the read side (vectorized pandas groupby, no per-row loop) consumed by `signals/news_catalyst.py::NewsCatalystSignal.pre_compute()`, which blends the resulting credibility-weighted social aggregate with the existing Finnhub-headline score per `settings.SENTIMENT_SOCIAL_BLEND_WEIGHT` (weights sum to 1.0; degrades to headline-only with no social documents). Module-level singleton via `get_sentiment_source()` / `reset_sentiment_source()`, mirroring `data/market_data.py`'s `get_provider()`/`reset_provider()`. **New env vars**: `SENTIMENT_INGESTION_ENABLED`, `SENTIMENT_SOURCES`, `SENTIMENT_INGESTION_LOOKBACK_DAYS`, `SENTIMENT_DESENTENCIZE_ENABLED`, `SENTIMENT_MAX_DOCUMENTS_PER_CYCLE`, `SENTIMENT_INGESTION_MAX_SECONDS_PER_CYCLE`, `SENTIMENT_CIRCUIT_BREAKER_THRESHOLD`, `SENTIMENT_SOCIAL_BLEND_WEIGHT`, `REDDIT_CLIENT_ID`/`REDDIT_CLIENT_SECRET`/`REDDIT_USER_AGENT`, `EDGAR_USER_AGENT`, `REDDIT_BACKFILL_MAX_PAGES` (all secrets in `SECRET_KEYS`, never GUI-writable), plus the non-secret `GOOGLE_NEWS_LOOKBACK_WINDOW` (str, default `"7d"`; no effect until `"google_news"` is added to `SENTIMENT_SOURCES`). **EDGAR full-text search (EFTS) extension (2026-07):** `EdgarSource` gained an opt-in additions path layered on top of (never replacing) the always-on 8-K submissions-feed logic above ‚Äî completely inert, byte-identical existing behavior, until `settings.EDGAR_FULLTEXT_ENABLED` (default `False`) is set. When enabled, `EdgarSource._fetch_fulltext()` calls SEC's full-text search index at `https://efts.sec.gov/LATEST/search-index` (path casing matters ‚Äî lowercase `/latest/` 404s), filtered by `settings.EDGAR_FULLTEXT_FORMS` (comma-separated, default `"8-K,10-K,10-Q"`, mirroring the `SENTIMENT_SOURCES` fan-out convention), the ticker's CIK (reusing the existing `_lookup_cik` cache), and a `since`-derived date range. Matched **10-K/10-Q** filings have their primary document fetched and converted to plain text (`_extract_filing_text` ‚Äî a best-effort `Item 1A`/`Item 7` Risk-Factors/MD&A regex narrows the text when those markers are cleanly present, but always falls back to the filing's FULL text rather than risk silently dropping content ‚Äî a documented, deliberate accuracy/robustness tradeoff, not a bug), then split into ‚â§`settings.EDGAR_FULLTEXT_CHUNK_TOKENS` (default 512, matching FinBERT's context window) whitespace-token chunks via `_chunk_text`. Matched filings of other configured form types (e.g. 8-K, when included in `EDGAR_FULLTEXT_FORMS`) are represented by a short entity/form headline-style string rather than a body fetch, since the always-on 8-K path above already covers 8-K headline text. Every chunk/headline-string from one `fetch()` call is scored TOGETHER via `signals/news_catalyst.py::score_headlines()` ‚Äî PR417's batched entry point ‚Äî rather than one call per chunk. **Leakage-safe timestamping**: every resulting document's `as_of` is derived from EFTS's own official `file_date` response field (never wall-clock fetch time), so a filing can never be attributed to the wrong trading day regardless of when the ingestion process happened to run ‚Äî proved by a dedicated test in `tests/test_sentiment_sources.py::TestEdgarFullTextSearch` mirroring `tests/test_sentiment_pit_lookahead.py`'s style. **Shared SEC rate-limit throttle**: `_efts_search`/`_fetch_filing_text` reuse `data/edgar_fundamentals.py`'s exact `_throttle()` (monotonic-clock sliding gap held across a lock, ~10 req/s) rather than a second, independent throttle, so this module's EDGAR full-text traffic shares the same process-wide SEC fair-access budget as the fundamentals backfill instead of each independently assuming it owns the full 10 req/s. Every failure (EFTS network/HTTP error, malformed JSON, a filing whose body text can't be fetched, a throttle/import error) degrades to skipping just that piece of work ‚Äî never raises, never discards the always-on 8-K docs a `fetch()` call already collected (CONSTRAINT #6). **New env vars**: `EDGAR_FULLTEXT_ENABLED` (bool, default `False`), `EDGAR_FULLTEXT_FORMS` (str, default `"8-K,10-K,10-Q"`), `EDGAR_FULLTEXT_CHUNK_TOKENS` (int, default `512`) ‚Äî all three added by the settings-scaffolding PR (#416) and wired by this extension. **Historical backfill (2026-07):** `GDELTSource.fetch()` was rewritten from a single "50 most recent articles" call (which would have silently returned only today's newest headlines for any backfill request, never genuine history) to `_MAX_WINDOWS`-bounded `_CHUNK_DAYS`-wide `startdatetime`/`enddatetime`-scoped windowed queries, so a `since` far in the past (e.g. a 5-month backfill) actually walks real historical windows; a live per-cycle call with a recent `since` still resolves in exactly one window/request. `RedditSource.fetch()` similarly moved off a hardcoded `t=day` to `_time_bucket_for(since)` (the narrowest Reddit `t=` bucket ‚Äî hour/day/week/month/year/all ‚Äî that still covers `since`) plus `after`-cursor pagination bounded by `REDDIT_BACKFILL_MAX_PAGES`, stopping as soon as a page's oldest post predates `since` (Reddit returns `sort=new`, so everything past that point is even older). GDELT/EDGAR/Finnhub are genuinely, honestly backfillable with **zero credibility bias** (institutional sources are policy-trusted at `1.0` regardless of when they're scored); Reddit backfill is real but carries a documented caveat ‚Äî a backfilled post's `S_authority` reflects the author's CURRENT account state, not their state at post time, since Reddit's API has no way to answer that (see `RedditSource`'s docstring). Yahoo RSS has no historical archive at all (a live feed only) and cannot backfill under any settings. `HistoricalStore.get_sentiment_archive_depth_by_source()` (grouped `MIN`/`MAX`/`COUNT` SQL aggregation, read-only) reports per-source archive depth so a future validation gate can weigh institutional depth separately from Reddit's rather than one blended `SENTIMENT_PIT_MIN_MONTHS` number that would overstate confidence in the weaker component. See `scripts/backfill_sentiment_history.py` in `docs/architecture/webapp-and-gui.md` for the operator-facing CLI.
- **Sector Heat Factor** (`data/sentiment_sources.py::GDELTVolumeSource`/`compute_sector_heat_factors`, follow-on to the PR #416 `Sector_Heat_Factor` schema scaffolding) ‚Äî a GDELT article-**volume** (not tone) attention proxy, aggregated per SECTOR (not per ticker ‚Äî bounded to the small number of distinct GICS sectors present in the universe each cycle, never by ticker count) and Gaussian-smoothed (`scipy.ndimage.gaussian_filter1d`, `sigma=settings.SECTOR_HEAT_SMOOTHING_SIGMA`) before "today's" value is surfaced as the `Sector_Heat_Factor` column. Deliberately NOT a `SentimentSource` subclass (produces a count time series, not `SentimentDocument`s) and NOT wired into `signals/`/`settings.SIGNAL_WEIGHTS`/scoring ‚Äî a feature column only, for now. `GDELTVolumeSource.fetch_daily_counts()` issues exactly ONE `mode=timelinevol` GDELT call per sector per cycle (returns the whole `settings.SECTOR_HEAT_LOOKBACK_DAYS`-day window in one response, unlike `GDELTSource`'s windowed `mode=artlist` backfill above), causal by construction (`enddatetime` is the query's own cutoff, plus a defensive drop of any point dated after it). Master gate `settings.SECTOR_HEAT_ENABLED` (default `False`) ‚Äî `compute_sector_heat_factors()` returns `{}` with zero network calls when off, reproducing the exact pre-existing `NaN`-fill placeholder behavior. Wired into `dashboard_df` by `pipeline/production_steps.py::_apply_sector_heat_factor()` (maps a `{sector: heat_value}` dict onto every ticker row via its `sector` column; any sector absent from the dict ‚Äî failed query, or a ticker with no/`"Unknown"` sector ‚Äî stays `NaN`, never fabricated) and surfaced in `main_orchestrator.py::_write_state_snapshot()`'s per-signal `sector_heat_factor` key. Full rationale (Fu & Zhang 2024 news-volume "heat"), settings table, and failure-mode enumeration: [`docs/signals/sector_heat_factor.md`](../signals/sector_heat_factor.md).
- **data/attention_sources.py** (Wikipedia-pageviews investor attention, 2026-07, follow-on to PR #416/#417) ‚Äî Grounded in Da, Engelberg & Gao (2011), "In Search of Attention," *Journal of Finance* 66(5):1461-1499 (SVI spikes predict higher prices over the following ~2 weeks, with an eventual reversal). **Master switch: `settings.WIKIPEDIA_ATTENTION_ENABLED`, default `False`** ‚Äî the sole call site, `pipeline/production_steps.py`'s `StrategyEvalStep`, is a complete no-op (zero network calls, `Attention_Score` stays `NaN`) until an operator sets this `True`, matching this codebase's other opt-in-networked-feature convention (`SENTIMENT_INGESTION_ENABLED`, `SECTOR_HEAT_ENABLED`). PRIMARY mechanism is Wikipedia's official, free, no-auth Pageviews REST API (`WikipediaPageviewsSource`, structurally analogous to ‚Äî but NOT a subclass of ‚Äî `data/sentiment_sources.py`'s `SentimentSource`, since this is a numeric attention/volume score rather than document ingestion); ticker‚Üíarticle-title resolution is deliberately SIMPLE (try the ticker symbol itself, then fall back to a `company_name` string if the caller has one on hand from this cycle's `FundamentalDataDTO` ‚Äî no dedicated ticker‚Üícompany-name resolution service is built). `_abnormal_attention_score()` computes a DEG-style abnormal-attention transform, `log(1 + recent_3-day-mean) ‚àí log(1 + baseline_median)` over `settings.WIKIPEDIA_ATTENTION_LOOKBACK_DAYS` (default 30) of daily pageviews ‚Äî NaN (never a fabricated 0.0, CONSTRAINT #4) below `_MIN_BASELINE_DAYS` (7) of baseline history, and explicitly drops any row dated after the `as_of` cutoff (no-lookahead defense-in-depth, see `tests/test_attention_pit_lookahead.py`). **Google Trends via `pytrends` is explicitly OPTIONAL and NEVER load-bearing** (separate gate: `settings.PYTRENDS_ENABLED`, default `False`) ‚Äî `pytrends`'s underlying repo (`GeneralMills/pytrends`) was archived read-only in April 2025 and is well known to be rate-limit-fragile (frequent HTTP 429s even under light use, since it scrapes an undocumented Google endpoint rather than a real API);
