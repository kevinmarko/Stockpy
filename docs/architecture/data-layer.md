# Architecture: Data Layer & Persistence

> Part of the split-out `CLAUDE.md` System Architecture reference. See [../../CLAUDE.md](../../CLAUDE.md) for the index.

- **config.py** — Single Source of Truth (SSOT). `COLUMN_SCHEMA` defines every column's Google Sheets header, internal dict key, and display format, plus Pandera schemas for validation. Any new field must be added here first before use elsewhere.
- **dto_models.py** — `MarketBarDTO`, `FundamentalDataDTO`, `MacroEconomicDTO`, `RobinhoodPositionDTO`. All market/fundamental/macro data flowing through calculation layers must be coerced into these DTOs — raw dict lookups or positional indexing in calc code is not allowed. DTO parsers also handle messy upstream strings (currency symbols, `%`, `"N/A"`, padding). `FundamentalDataDTO` carries a `current_ratio` field (`__init__` default `NaN`; `from_raw_dict` reads `info.get("currentRatio", NaN)`) — `currentRatio` is already emitted by `data/yahoo_fundamentals.py` and is now carried through the DTO and surfaced by `engine/advisory.py` as `key_indicators["current_ratio"]`. `MacroEconomicDTO` now accepts an optional `hmm_risk_on_probability: Optional[float] = None` (the `regime/hmm_regime.py` second opinion). `None` (the default) reproduces the exact pre-HMM behavior. The rules-based `_rules_based_regime` stays primary; `market_regime` downgrades RISK ON → NEUTRAL (logged) when `hmm_risk_on_probability < HMM_RISK_ON_DOWNGRADE_THRESHOLD` (0.3) — it can never independently declare a worse regime or upgrade a better one. `killSwitch` ORs in a lowered-threshold check (`vix > KILLSWITCH_VIX_THRESHOLD_AGREED`=25 or `sahm_rule_indicator >= KILLSWITCH_SAHM_THRESHOLD_AGREED`=0.3, vs. the base 30/0.5) only when the rules-based regime is RECESSION AND the HMM agrees (`1 - hmm_risk_on_probability > HMM_RISK_OFF_AGREEMENT_THRESHOLD`=0.7) — strictly OR'd with the base condition, never less sensitive.
- **data_engine.py** — `DataEngine` (live Yahoo Finance/FRED via `fredapi`) and `MockDataEngine` (test fixture), both implementing an `IDataProvider` interface. Fetching is strictly decoupled from calculation. `IDataProvider` now also declares `fetch_macro_history() -> pd.DataFrame` (full historical VIX/yield-curve series for `regime/hmm_regime.py`'s expanding-window fit, distinct from `fetch_macro_raw()`'s single current-snapshot dict). `DataEngine.fetch_macro_history()` pulls full unbounded series via `self.fred.get_series('VIXCLS'/'T10Y2Y')`; returns an empty DataFrame (never fabricated rows) if FRED is unavailable. `MockDataEngine.fetch_macro_history()` returns a deterministic 500-row synthetic series (seeded) for offline tests. **Parallel fetch (2026-07):** `DataEngine.fetch_technical_raw()`/`fetch_fundamentals_raw()` were serial `for symbol in tickers:` loops making one blocking yfinance HTTP call at a time; both now route through a `ThreadPoolExecutor` sized by `settings.DATA_FETCH_MAX_CONCURRENCY` (default 8, matching the `FORECAST_MAX_CONCURRENCY`/`ADVISORY_MAX_CONCURRENCY` convention), collapsing wall-clock fetch time from ~N sequential round-trips to ~N/workers. Per-ticker try/except is preserved (one bad symbol still can't abort the batch); the old `sleep(0.1)`-every-5-tickers throttle in `fetch_fundamentals_raw` was dropped in favor of the bounded worker count as the de-facto rate limit. Set `DATA_FETCH_MAX_CONCURRENCY=1` to force the original sequential path. **New env var:** `DATA_FETCH_MAX_CONCURRENCY`. **`fetch_technical_raw_cached()` — HistoricalStore routing for `main_orchestrator.py` (2026-07):** `main_orchestrator.py`'s `fetch_all_data_async()` was the last major tech-bars call site that bypassed `HistoricalStore` entirely — it called `DataEngine.fetch_technical_raw()` directly every cycle for the full ticker universe (unlike `main.py`'s `_fetch_bars_for_universe()`, which already routes through `HistoricalStore.get_bars()`). `DataEngine.fetch_technical_raw_cached(tickers)` closes this gap: a new, additive method (alongside, not replacing, `fetch_technical_raw()`) that lazily imports `data.historical_store.HistoricalStore` and `data.market_data.get_provider()` (this repo's `HistoricalStore`-lazy-import convention, matching `processing_engine.py`/`macro_engine.py`) and routes each ticker through `HistoricalStore.get_bars(symbol, lookback_days=settings.BARS_BACKFILL_DAYS, provider=<CompositeProvider singleton>)` instead of a fresh `yf.Ticker(symbol).history(period="2y")` call — a symbol whose bars are already persisted needs only its `(last_date, today]` delta fetched. Preserves the exact `ThreadPoolExecutor`/`DATA_FETCH_MAX_CONCURRENCY` concurrency pattern and per-ticker try/except dead-letter resilience of `fetch_technical_raw()`. Falls through to calling `fetch_technical_raw(tickers)` verbatim (byte-identical output) when `settings.HISTORICAL_STORE_ENABLED` is `False`, or when `HistoricalStore`/`get_provider()` construction raises — CONSTRAINT #6, this can only ever remove a redundant refetch, never silently substitute bad data. `MockDataEngine` gained a matching `fetch_technical_raw_cached()` that is a plain alias for its own `fetch_technical_raw()` (mock data is synthesized fresh from `datetime.now()` on every call, so there is no real per-ticker fetch to cache against a DB; this also keeps `main_orchestrator.py`'s existing `MockDataEngine()` offline-fallback branches — no `credentials.json`, or a live-fetch coming back empty — working unchanged now that the call site is unconditional). `fetch_technical_raw()` itself is unchanged.
- **data/market_data.py** — Swappable market-data layer with a provider abstraction (`MarketDataProvider` ABC) that hides the concrete backend from all signal, indicator, and forecasting code. **Provider auto-selection** (evaluated at `CompositeProvider` construction): (1) `MARKET_DATA_PROVIDER=alpaca` or Alpaca keys present → `AlpacaProvider` (real-time IEX via alpaca-py); (2) otherwise → `YFinanceProvider` (zero config, ~15-min delayed, `is_stale=True` always). **Components**: `Quote` (frozen dataclass: symbol, price, bid, ask, UTC timestamp, is_stale, source); `MarketDataError` (typed exception — orchestrator catches per-symbol for dead-letter resilience); `AlpacaProvider` (`StockHistoricalDataClient` + `StockLatestQuoteRequest` + `StockBarsRequest`, IEX feed, stale if quote age > `stale_threshold_seconds`=60); `YFinanceProvider` (`Ticker.fast_info` for quotes, `Ticker.history()` for bars — always `is_stale=True`, wraps yfinance errors as `MarketDataError`; ALSO the fundamentals fallback, returning raw `Ticker.info` when the primary Yahoo-computed engine yields nothing); `YahooFundamentalsProvider` (`SOURCE="yahoo_computed"` — the **PRIMARY fundamentals source**; pulls Yahoo's FREE financial statements — `income_stmt`, `balance_sheet`, `cashflow`, `dividends`, `institutional_holders` — plus price/shares, then delegates to the pure `data/yahoo_fundamentals.py::compute_fundamentals(...)` to derive 15 equity fundamentals as a yfinance `.info`-style dict so `FundamentalDataDTO.from_raw_dict()` is unchanged; NaN-degrading per CONSTRAINT #4 — missing input → NaN, never a fabricated 0.0; caches SPY returns for the beta computation); `FinnhubProvider` (**DEPRECATED as a fundamentals source and UNWIRED from `CompositeProvider`** — the class is retained but no longer consulted for fundamentals; `FINNHUB_API_KEY` is now used ONLY by `signals/news_catalyst.py` for company news / earnings); `CompositeProvider` (routes quotes/bars to the selected backend, fundamentals to **`YahooFundamentalsProvider` (primary) → yfinance `.info` (emergency fallback)** — Finnhub is no longer in the fundamentals path; exposes a `source_name` property reporting the active fundamentals backend; in-process TTL quote cache, logs startup banner once). **Bar shape contract**: `get_intraday_bars()` returns a DataFrame with columns `Open, High, Low, Close, Volume` and a timezone-naive `DatetimeIndex` sorted ascending — identical to `DataEngine.fetch_technical_raw()`'s shape so all processing/forecasting/strategy code runs unchanged. **Quote cache**: in-process dict + monotonic TTL (default `MARKET_DATA_QUOTE_TTL_SECONDS`=30 s), never written to disk. **Bars cache (2026-07, PR A hot-path)**: `CompositeProvider.get_intraday_bars()` also carries an in-process `_BarsCache` (thread-safe `threading.Lock`, keyed `(symbol, lookback_days)` → `(df, monotonic-expiry)`, default `MARKET_DATA_BARS_TTL_SECONDS`=300 s, returns a defensive `df.copy()` so callers can't poison the cache) so the "bars fetched twice per symbol" pattern (universe pre-fetch + advisory-overlay refetch) collapses to one backend pull within the TTL — bars are daily-resolution so a few-minutes TTL is safe. `invalidate_bars(symbol, lookback_days)` / `clear_bars_cache()` force a refresh. **Two SCALE-CRITICAL emission rules (do NOT "fix" — downstream depends on them)**: `dividendYield` is emitted as a **FRACTION** (e.g. `0.0257`, not `2.57`) — it is NOT routed through `normalize_yfinance_dividend_yield` and is consumed as-is; `debtToEquity` is emitted as a **PERCENT** (`×100` → e.g. `150.0`, not `1.5`) because two downstream consumers divide by 100. Plain-fraction ratios (`returnOnEquity`, `grossMargins`, `operatingMargins`, `payoutRatio`) are unscaled; `payoutRatio` uses `abs()` of the (negative) "Cash Dividends Paid" cash-flow line. **Beta** = `Cov(stock, SPY) / Var(SPY)` over `BETA_LOOKBACK_DAYS` (default 504) of daily returns, with the SPY return series cached inside the provider. **Fundamentals cache**: `CompositeProvider` carries an in-process `_FundamentalsCache` (default `FUNDAMENTALS_CACHE_TTL_SECONDS`=21600 s / 6 h) that caches positive AND empty (negative) responses so coverage-miss symbols never re-hit the network within the TTL window. **Finnhub 429 machinery (retained but dormant for fundamentals)**: the deprecated `FinnhubProvider` still wraps outbound calls in a `_SlidingWindowRateLimiter` (default `FINNHUB_RATE_LIMIT_PER_MIN`=50) with one-shot exponential backoff + 429 retry — relevant now only if a caller invokes it directly, since it is no longer on the fundamentals path. `clear_fundamentals_cache()` purges the cache for forced refresh. Module-level singleton via `get_provider()` / `reset_provider()`. **New env vars**: `MARKET_DATA_PROVIDER`, `FUNDAMENTALS_SOURCE` (`"yahoo"` default | `"yfinance_info"`), `BETA_LOOKBACK_DAYS`, `FINNHUB_API_KEY` (now `news_catalyst`-only, NOT a fundamentals dependency), `MARKET_DATA_QUOTE_TTL_SECONDS`, `MARKET_DATA_BARS_TTL_SECONDS` (bars-cache TTL, default 300 s, GUI-writable non-secret allowlist key), `FUNDAMENTALS_CACHE_TTL_SECONDS`, `FINNHUB_RATE_LIMIT_PER_MIN`.
- **data/sentiment_sources.py** (Sentiment Pipeline Phase 3, 2026-07) — Multi-source sentiment-ingestion provider abstraction, structurally mirroring `data/market_data.py`'s `MarketDataProvider` ABC but **fanning out** across every enabled source rather than selecting one backend, since each source contributes independent documents. `SentimentSource` ABC (`fetch(symbol, since) -> List[SentimentDocument]`, must return `[]` never raise); concrete free-first providers `FinnhubSentimentSource` (wraps `signals/news_catalyst.py`'s existing client), `YahooRSSSource`, `GDELTSource`, `RedditSource` (OAuth2 client-credentials, degrades to `[]` without `REDDIT_CLIENT_ID`/`REDDIT_CLIENT_SECRET`), `EdgarSource` (SEC full-text 8-K filings by CIK lookup, degrades to `[]` without a compliant `EDGAR_USER_AGENT`). `SentimentDocument` (frozen dataclass) carries optional credibility-relevant raw inputs (`author_handle`, `author_followers`, `account_age_days`) — `None` when the source doesn't expose them, never fabricated (CONSTRAINT #4). `CompositeSentimentSource` fans out per `settings.SENTIMENT_SOURCES` (comma-separated, default `"finnhub,yahoo_rss,gdelt,reddit,edgar"`), dedupes via a rolling hash keyed on `(source, symbol, trading_day, text)`, applies the optional `desentencize()` FinBERT preprocessing trick (period→semicolon, protecting decimals and abbreviations; gated by `SENTIMENT_DESENTENCIZE_ENABLED`, default `False`), and enforces a per-cycle document budget (`SENTIMENT_MAX_DOCUMENTS_PER_CYCLE`, default 2000) as its bounded, in-process backpressure mechanism — sources are polled in priority order (`finnhub, edgar, yahoo_rss, gdelt, reddit`) and lower-priority sources are shed once the budget is hit, **never** touching order/broker code under any pressure condition. `fetch_and_archive()` persists every document to `data/historical_store.py`'s `sentiment_ingestion_audit` table (gated by `SENTIMENT_AUDIT_ENABLED`) via the shared `HistoricalStore.resolve_trading_day()` UTC→ET post-close roll. Module-level singleton via `get_sentiment_source()` / `reset_sentiment_source()`, mirroring `data/market_data.py`'s `get_provider()`/`reset_provider()`. **New env vars**: `SENTIMENT_SOURCES`, `SENTIMENT_INGESTION_LOOKBACK_DAYS`, `SENTIMENT_DESENTENCIZE_ENABLED`, `SENTIMENT_MAX_DOCUMENTS_PER_CYCLE`, `REDDIT_CLIENT_ID`/`REDDIT_CLIENT_SECRET`/`REDDIT_USER_AGENT`, `EDGAR_USER_AGENT` (all secrets in `SECRET_KEYS`, never GUI-writable).
- **data/robinhood_client.py** — `RobinhoodClient` API wrapper (`robin_stocks`). Logs in (falling back to interactive terminal input if MFA is required), fetches positions, unzips average costs and shares, and merges them into `main.py` and `main_orchestrator.py` universes. **Task 1.4 discovery helpers:** module-level `discover_watchlists(client) -> {watchlist_name: [tickers]}` reads every Robinhood "Lists" entry via `r.get_all_watchlists()` + `r.get_watchlist_by_name()` (per-list failures logged & skipped — never raises). `discover_universe(client, extra_files=None) -> list[str]` returns one sorted/deduped universe of holdings ∪ every RH watchlist ∪ plain-text files supplied via the `extra_files` argument or the colon-separated `SYNC_WATCHLIST_FILES` env var. Both helpers short-circuit cleanly when the client is not authenticated. **2026-06 noise suppression**: Robinhood's `midlands/lists/items/?list_id=<UUID>` endpoint returns 400 for certain system-curated watchlists; `robin_stocks` prints the HTTPError via `print(message, file=helper.get_output())` (not `logging`), which floods stdout on every account. Both discovery calls (`get_all_watchlists`, `get_watchlist_by_name`) are now wrapped in the `_suppress_rs_output()` context manager that swaps `robin_stocks.robinhood.helper`'s output sink to an in-memory `StringIO` for the duration of the call and forwards any captured text to the module logger at DEBUG, then restores the prior sink. Functional behavior is unchanged — the 400-empty result still degrades silently via the existing `or []` and try/except, only the stdout spam is gone.
- **data/portfolio_sync.py** — **Task 1.4 Portfolio & Watchlist Synchronization Engine.** Public API: `CoverageStatus` enum (`FULL` / `QUOTES_ONLY` / `EQUITY_ONLY` / `UNCOVERED` / `UNKNOWN`), frozen dataclasses `SymbolStatus` and `SyncReport`, `build_sync_report(snapshot, *, client=None, watchlist_files=None, forecast_symbols=None, probe_market=True) -> SyncReport`, async wrapper `async_sync_now(...)` (used by the GUI Live Inventory "Sync Now" button), and `write_cache`/`read_cache` for the JSON snapshot at `cache/sync_report.json` (atomic write-then-rename). `build_sync_report` (a) takes the union of holdings (Robinhood `AccountSnapshot.positions`) + every Robinhood watchlist + file-backed lists; (b) probes each symbol against `data.market_data.get_provider()` for quote+bar+fundamentals coverage (per-symbol try/except, never aborts); (c) classifies each symbol — held symbols whose market-data probe fails are **upgraded** from `UNCOVERED` to `EQUITY_ONLY` so they remain visible in the equity view while being excluded from pricing-dependent metrics; (d) emits one structured INFO log per coverage gap (non-blocking diagnostic per the Task 1.4 spec). **No fabricated metrics** (CONSTRAINT #4): a held position with no live quote and a non-positive Robinhood `market_value` reports `current_price=NaN`, `market_value=NaN`; `SyncReport.held_total_equity()` then falls back to `qty * avg_cost` so the cost-basis-anchored equity view stays accurate without inventing a current-price proxy. `async_sync_now(persist_default_tickers=True)` runs `build_sync_report` off-thread and writes the resulting sorted universe to `DEFAULT_TICKERS` in `.env` via `gui.env_io.write_setting` (the allowlist-bounded path); a `SecretWriteError`/`DisallowedKeyError` from env_io is caught and logged rather than propagated so the GUI's refresh handler never crashes. **New env var:** `SYNC_WATCHLIST_FILES` (colon-separated paths to additional plain-text watchlist files; missing files tolerated silently).
- **data/historical_store.py** — **Tier 2.3 Phase 1 + Phase 2 + Phase 3.** Persistent OHLCV bar cache, Robinhood account snapshot store, fundamentals history, and FRED macro series backed by `quant_platform.db` (raw `sqlite3` + WAL). Public API: `HistoricalStore(db_path="quant_platform.db")` — creates all tables lazily (price_bars, account_snapshots/positions, fundamentals_history, macro_history, news_history, and — Sentiment Pipeline Phase 2, 2026-07 — `sentiment_ingestion_audit`, a per-DOCUMENT sentiment audit trail distinct from `news_history`'s per-symbol daily aggregate; no FK on `symbol`, see `save_sentiment_documents()` and the leakage-critical `resolve_trading_day()` UTC→America/New_York post-close roll, reused by `data/sentiment_sources.py`). **Phase 1 — price_bars**: `get_bars(symbol, lookback_days=504, *, provider=None) -> pd.DataFrame` returns tz-naive `DatetimeIndex` with columns `[Open, High, Low, Close, Volume]` — identical to `DataEngine.fetch_technical_raw()`; `latest_bar_date(symbol) -> Optional[pd.Timestamp]`. Incremental rule: `SELECT MAX(date)` → fetch `(max_date, today]` only; first call = `BARS_BACKFILL_DAYS` (504) full backfill. **Market Holiday Defense (2026-07)**: `_get_bars_db_path` checks for weekends and holidays using `pandas.tseries.holiday.USFederalHolidayCalendar`. If no trading days have elapsed since the last stored bar date, it returns cached data directly and skips calling the live provider (avoiding empty fetches). **Phase 2 — account_snapshots / account_positions**: `save_account_snapshot(snapshot) -> int` (returns `snapshot_id` or `-1` on error, single ROLLBACK-safe transaction); `latest_account_snapshot() -> Optional[AccountSnapshot]` (reconstructs full dataclass + positions dict from DB); `account_snapshot_history(since=None) -> pd.DataFrame` (equity-curve data). **Phase 3 — fundamentals_history + macro_history**: `get_fundamentals(symbol, max_age_days=1, *, provider=None) -> Dict[str, float]` returns typed columns `{pe_ratio, pb_ratio, roe, dividend_yield, market_cap, eps, operating_margin, debt_to_equity}` — missing fields are `NaN`, NEVER `0.0` (CONSTRAINT #4); caches the full `raw_json` provider dict for future PIT-fundamentals replay; daily TTL controlled by `FUNDAMENTALS_REFRESH_DAYS`. `get_fundamentals_history(symbol, since=None) -> pd.DataFrame` returns all accumulated snapshots for PIT replay (see note below). `get_macro(series_id, *, lookback_days=None, data_engine=None) -> pd.Series` returns a tz-naive DatetimeIndex Series from `macro_history`, topping up from FRED via `data_engine.fetch_macro_history()` when the cached rows are older than `MACRO_REFRESH_HOURS` (default 12 h). All methods dead-letter resilient (try/except → empty sentinel, never raise — CONSTRAINT #6). No fabricated data (CONSTRAINT #4). `AccountSnapshot` is the in-memory source of truth; DB tables are derived from it. **Connection reuse (2026-07, PR A hot-path)**: instead of opening a fresh `sqlite3.connect(...)` (+ WAL/busy_timeout PRAGMAs) on every call — ~N-per-ticker/cycle before — the store now holds ONE cached connection (`self._conn` via `_get_conn()`, `check_same_thread=False`, WAL + `busy_timeout=5000`) guarded by `self._lock` (`threading.Lock`) so it is safe to share across the orchestrator's now-parallelized advisory loop; `_ensure_tables()` still uses a short-lived connection so `sqlite3.connect`-patch degradation tests keep passing. **Settings**: `HISTORICAL_STORE_ENABLED` (default `True`), `BARS_BACKFILL_DAYS` (default 504), `FUNDAMENTALS_REFRESH_DAYS` (default 1), `MACRO_REFRESH_HOURS` (default 12). **PIT-fundamentals note**: `fundamentals_history.raw_json` accumulates real point-in-time snapshots from the day Phase 3 ships. After ≥ 90 days of history, `get_fundamentals_history()` can be used to extend the multifactor validation harness to the Value/Quality factors — see `tests/test_validation_multifactor.py` docstring for the extension path (out-of-scope for Phase 3). **`get_fundamentals_raw(symbol, max_age_days=1, *, provider=None) -> Dict[str, Any]`**: unlike `get_fundamentals()` (which returns only the eight typed columns above), this returns the FULL raw provider dict — the shape `FundamentalDataDTO.from_raw_dict()` needs (`sector`, `company_name`, `book_value`, `payout_ratio`, `dividend_growth_rate`, `current_ratio`, etc.), which the typed columns alone would silently drop. On a fresh cache hit it parses the already-stored `raw_json` column directly (zero network call, reusing `_read_fundamentals_row()`); on stale/missing it fetches live and persists via the SAME `_upsert_fundamentals()` write path `get_fundamentals()` uses, so both methods' caches stay consistent. `{}` on total failure (CONSTRAINT #6).
- **data/robinhood_portfolio.py** — **ADVISORY ONLY. NO ORDER CODE.** Read-only Robinhood account snapshot via TOTP MFA (`pyotp`). Public API: `fetch_account_snapshot(max_age_hours=20.0, force=False) -> AccountSnapshot` and `logout()`. Frozen dataclasses: `PortfolioPosition` (symbol, qty, avg cost, current price, market value, unrealized P/L, dividends received, name) and `AccountSnapshot` (positions dict, buying\_power, total\_equity, total\_dividends, fetched\_at UTC-aware datetime). `AccountSnapshot.age_hours()` / `is_stale(max_age_hours)` surface staleness. **Three-tier read order (Tier 2.3 Phase 2):** (1) `HistoricalStore.latest_account_snapshot()` — DB, fastest, no network; (2) `cache/account_snapshot.json` — daily JSON cache, secondary fallback; (3) live Robinhood fetch. After a successful live fetch, the snapshot is written to BOTH the JSON cache AND the DB. Stale cache is returned (not errored) on live-fetch failure; raises only when live fails and no cache exists. Username/Password variables are read from `os.environ` (`RH_USERNAME`/`RH_PASSWORD`). If `RH_MFA_SECRET` is unset or empty, login falls back to prompting for MFA interactively. No secrets are ever written to any cache payload. **New adapter:** `account_snapshot_to_robinhood_positions(snapshot: Optional[AccountSnapshot]) -> Dict[str, RobinhoodPositionDTO]` bridges the rich `PortfolioPosition` shape to the leaner `dto_models.RobinhoodPositionDTO` (`ticker`, `shares`, `average_cost`, `total_dividends`) that `main_orchestrator.py`'s `run_pipeline(robinhood_positions=...)` parameter expects — a pure field-mapping function (quantity→shares, average_cost→average_cost, dividends_received→total_dividends), no network calls, and an empty/`None` snapshot degrades to `{}` rather than raising (CONSTRAINT #6). **Brokerage-connect credential verification (2026-07, one-time-code redesign):** `_login()`'s inner `r.login()` call is factored into a shared `_login_with(username, password, mfa_code, *, allow_interactive=False)` helper that takes an ALREADY-GENERATED 6-digit MFA code — it has no `pyotp`/TOTP-secret knowledge of its own. `_login()` (the env-var path used by every read pipeline call) derives that code itself from `RH_MFA_SECRET` via `pyotp.TOTP(...).now()` before calling `_login_with`, preserving its exact prior external behavior (env-var read, interactive MFA fallback when unset). `verify_credentials(username: str, password: str, mfa_code: str = "") -> bool` (public) is the ONLY other caller of `_login_with` — it takes credentials as EXPLICIT arguments (not `os.environ`), used exclusively by the `api/pilots_api.py::POST /brokerage/connect` intake flow to verify a candidate username/password plus the user's CURRENT one-time 6-digit authenticator-app code (never the underlying TOTP secret, which this flow never collects or sees) BEFORE the username/password are ever persisted. Unlike `_login()`, it NEVER falls back to interactive MFA prompting (`allow_interactive=False` — a headless HTTP request must not block on stdin; a missing/empty `mfa_code` is treated as a verification failure). Logs out immediately on a successful verification login (no lasting session beyond `robin_stocks`' own `store_session=True` device-pickle at `~/.tokens`, which is what lets *subsequent* unattended re-fetches through `_login()`'s own `RH_USERNAME`/`RH_PASSWORD`-only path keep working without another code, until that device session eventually expires). Never raises (any failure → `False`) and never logs credential values — only exception *type names* (CONSTRAINT #3).
- **data/brokerage_credentials.py** (2026-07, new; TOTP-secret scope dropped 2026-07) — **LOCAL, SINGLE-OPERATOR ONLY.** Dedicated, hard-scoped `.env` writer for Robinhood portfolio-snapshot credentials, used ONLY by `api/pilots_api.py::POST /brokerage/connect` (never by the GUI). Exists separately from `gui/env_io.py` because that module's whole purpose is to REFUSE to write anything in `SECRET_KEYS` (CONSTRAINT #3) — this module is the one deliberate, narrowly-scoped exception. Public API: `write_rh_credentials(username, password) -> None` (hard allowlist of exactly `{RH_USERNAME, RH_PASSWORD}` — no other key is ever touched; writes via `dotenv.set_key`/`unset_key` AND mirrors into the live process `os.environ` so the current process picks them up without a restart), `clear_rh_credentials() -> None` (idempotent, removes both from both `.env` and `os.environ`), `rh_credentials_present() -> bool` (checks `os.environ` only, never returns/logs the values). This module deliberately never reads, writes, or clears `RH_MFA_SECRET` — that key belongs solely to the main pipeline's own operator-set `.env` credential for `data/robinhood_portfolio.py`'s unattended login and must never be touched by this webapp-facing, one-time-code intake path (a webapp reconnect can never silently clobber it). Callers MUST have already verified credentials via `data.robinhood_portfolio.verify_credentials` (the one-time 6-digit code, never persisted) — this module performs no verification of its own, only persistence, and never logs credential values (only key names/lengths, matching `gui/env_io.write_setting`'s convention). Not a multi-user vault: one `.env` file, one machine, matching this codebase's local-first single-operator model (AGENTS.md §1).
- **data/robinhood_orders.py** — **ADVISORY ONLY. NO ORDER CODE.** Read-only realized-P&L engine (Tier 7). Fetches FILLED equity orders (`get_all_stock_orders`) and reconstructs closed round-trip trades via PURE FIFO lot-matching (`reconstruct_closed_trades`), then summarises realized P&L / win rate / profit factor / holding stats (`realized_pnl_summary`). `parse_orders(raw, symbol_resolver)` normalises raw order dicts (filled-only, instrument-URL → ticker); `fetch_filled_orders(...)` adds a daily `cache/robinhood_orders.json` cache + dead-letter resilience (injectable `orders_fetcher`/`symbol_resolver` for tests); `realized_performance(...)` is the fetch→reconstruct→summarise convenience. Excess/short sells drop the unmatched qty (no fabricated entry); empty summaries are NaN-shaped (CONSTRAINT #4). Analytics-only — does NOT auto-write the `trades` table.
